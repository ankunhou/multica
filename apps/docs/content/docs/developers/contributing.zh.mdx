## 开发模型

本地开发优先使用 `just`。`Justfile` 只保留命令入口，跨平台流程逻辑放在 `scripts/just.mjs`。

## 前置要求

- Node.js `v20+`
- `pnpm` `v10.28+`
- Go `v1.26+`
- Docker
- `just`

## 首次设置

```bash
cp .env.example .env
just setup
```

## 启动服务

启动后端和 Web：

```bash
just start
```

只启动单项服务：

```bash
just server
just web
```

## Desktop 开发

建议开三个终端：

```bash
just start
```

```bash
just build
just cli-setup
just daemon
```

```bash
just desktop
```

## 常用命令

```bash
just build
just typecheck
just test-ts
just test-go
just check
just sqlc
just migrate-up
just migrate-down
```

## 数据库

```bash
just db-up
just db-down
just db-reset
```

`just db-reset` 只允许对本地数据库运行，避免误删远程数据库。

## 自托管

```bash
just selfhost
just selfhost-build
just selfhost-stop
```

## 维护规则

- 不再新增旧式命令入口。
- 文档中的开发命令应以 `just --list` 为准。
- 新增跨平台命令时，优先使用 Node 原生代码实现。
