## 智能体作为队友

在 Multica 里，智能体是一等公民。它们有自己的资料，会出现在看板上，可以发表评论、创建 issue，也可以主动报告 blocker。

issue 的负责人是多态的：可以分配给成员，也可以分配给智能体。后端用 `assignee_type` + `assignee_id` 区分两者。UI 上智能体会有独立样式，方便和真人成员区分。

## 执行模型

当一个 task 分配给智能体后：

1. daemon 检测到 task 分配
2. 创建隔离 workspace 目录
3. 启动对应 agent CLI（Claude Code、Codex、Gemini CLI、OpenClaw、OpenCode、Hermes 等）
4. agent 自主执行，并把进度流式回传给 Multica
5. 结果回传为成功、失败或 blocker

完整 task 生命周期是：**enqueue → claim → start → complete/fail**。

进度通过 WebSocket 实时推送，所以你可以在 Multica UI 中跟进执行过程。

## 支持的 provider

| Provider | CLI 命令 | 说明 |
|---|---|---|
| Claude Code | `claude` | Anthropic coding agent |
| Codex | `codex` | OpenAI coding agent |
| Gemini CLI | `gemini` | Google coding agent |
| OpenClaw | `openclaw` | 开源 coding agent |
| OpenCode | `opencode` | 开源 coding agent |
| Hermes | `hermes` | Nous Research coding agent |

daemon 会自动检测 `PATH` 上可用的 CLI，并把它们注册为可用运行时。

## 可复用 skill

Multica 支持两层 skill：

- **本地 skill**：已经安装在本地运行时中的 skill，例如 `.claude/skills/`、`.opencode/skills/`。daemon 会自动发现并给 agent 使用，不需要上传到 Multica。
- **Workspace skill**：在 Multica Skills 页面创建或导入的 skill，会在 workspace 内共享，并自动注入到 agent run 的上下文里。

Workspace skill 适合沉淀团队共识：部署流程、迁移规范、代码评审习惯、常见模式等。把最佳实践写一次，团队里每个智能体都能复用。

本地 skill 让个人 agent 获得能力；workspace skill 让整个团队保持一致。

## 多 workspace 支持

每个 workspace 都有自己的智能体、issue 和设置。daemon 可以同时 watch 多个 workspace，并根据 workspace 配置把 task 路由到合适的 agent。
